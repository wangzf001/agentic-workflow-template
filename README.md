# Agentic Workflow Template v1.1

> **给 AI 看的团队搭建说明书。** 当你需要为任何领域搭建 Skill 驱动的 Agentic 团队系统时，直接用这个框架。

---

## 一、核心哲学

```
❌ 旧范式：AI 是一个工具，你告诉它做什么
✅ 新范式：AI 是一个团队，你告诉它管什么
```

把每个 Skill 当作一个人，有独立的职责、方法论、输入输出。管理 AI 就像管理团队：分工、授权、质检。

---

## 二、四层闭环架构

```
用户（CEO）
  │
  └─ 总监（调度者）
        │
   ┌────┼────┐
   ↓    ↓    ↓
 采集层  分析层  决策层  反馈层 ← 🆕
 (并行)  (并行)  (串行)  (事后)
```

| 层级 | 职责 | 执行模式 | 人数 |
|------|------|:--:|:--:|
| **采集层** | 从不同源头拉原始数据 | 并行 | 2-3 |
| **分析层** | 不同方法论独立出报告 | 并行 | 2-3 |
| **决策层** | 综合→风控→质检 | 串行 | 2-3 |
| **🆕 反馈层** | 结果验证→迭代修正→角色评估 | 事后串行 | 1-2 |

### 🆕 反馈层：系统越用越准的引擎

```
交付 → 结果发生（市场走完/发布数据回来/项目上线）
  ↓
┌──────────────────────────┐
│        🧠 反馈层          │
│                          │
│  📋 复盘师                │
│  ├─ 回捞预判：上次说了什么？
│  ├─ 对照结果：实际发生了什么？
│  ├─ 逐条标记：✅对 ❌错 🔄待定
│  ├─ 归因分析：为什么错？忽略了什么？
│  └─ 输出「迭代日志」
│                          │
│  📊 绩效追踪师             │
│  ├─ 更新每个角色的「能力卡」
│  ├─ 追踪胜率/偏差/趋势
│  └─ 输出「权重建议」→ 下次策略师用
│                          │
│  产出：迭代日志 + 角色能力卡 + 权重建议
└──────────────────────────┘
  ↓
下次运行自动加载 → 策略师加权时引用历史胜率
```

#### 复盘师工作流

```
Step 1: 回捞上次预判
  从执行日志中读取上一次完整模式的策略师输出
  提取：方向判断、关键假设、风险提示

Step 2: 采集实际结果
  拉取同期实际数据（股价走势/发布数据/项目结果）

Step 3: 逐条标记
  | 预判 | 结果 | 标记 | 归因 |
  |------|------|:--:|------|
  | 胜宏340有支撑 | 最低到321后反弹 | ✅ | 支撑位判断偏保守 |
  | 科创50有望突破2200 | 收在2168 | ❌ | 忽略了北向流出 |
  | 光模块利好兑现 | 确实涨了4% | ✅ | — |

Step 4: 提炼迭代
  至少输出1条可操作的改进，写入迭代日志：
  "策略师对支撑位的判断偏保守(偏差均值-5%)→下次放宽支撑区间(±3%)"
  "北向资金权重在快速模式中被低估→快速模式加回资金采集员"
```

#### 绩效追踪师：角色能力卡

每个分析和决策角色拥有一张持久能力卡，跨会话保存：

```json
{
  "量化分析师": {
    "total": 47, "correct": 32, "win_rate": 0.68,
    "bias": "+8%",           // 倾向高估 8%
    "best_dim": "超跌检测",  // 最强维度
    "worst_dim": "政策因子", // 最弱维度
    "trend": "improving",    // 近10次 8/10
    "weight": 1.2            // 策略师加权系数
  },
  "基本面分析师": {
    "total": 23, "correct": 14, "win_rate": 0.61,
    "bias": "-4%",           // 倾向保守
    "best_dim": "风险识别",
    "worst_dim": "估值判断",
    "trend": "stable",
    "weight": 1.0
  },
  "产业链分析师": {
    "total": 15, "correct": 12, "win_rate": 0.80,
    "bias": "0%",
    "best_dim": "硬卡口识别",
    "worst_dim": "短期择时",
    "trend": "improving",
    "weight": 1.3
  }
}
```

#### 权重自适应

策略师做综合研判时，不再是等权平均三份报告，而是：

```
默认权重：量化 0.33 | 基本面 0.33 | 产业链 0.33

每次复盘后，绩效追踪师根据最近10次表现调权：
  产业链 近10次 9/10 → 权重 0.40
  量化   近10次 7/10 → 权重 0.35
  基本面 近10次 6/10 → 权重 0.25

调权规则：
  - 胜率差 <10% → 不调权（等权）
  - 胜率差 10-20% → ±0.05
  - 胜率差 >20% → ±0.10
  - 单角色权重上限 0.50（防过度集中）
  - 任何角色 <15次预测 → 不参与加权（样本太小）
```

> **核心原则**：第一周你是 CEO 凭直觉管团队。三个月后，数据告诉你该听谁的。

## 三、🆕 三档运行模式

不是所有问题都需要全员出动。按场景分级：

| 模式 | 触发条件 | 出动角色 | 输出 | 耗时 |
|------|------|------|------|:--:|
| **⚡ 预警模式** | 定时监控/阈值触发 | 采集层(1人)+风控官 | 一句话预警 | <5s |
| **🚀 快速模式** | 日常扫盘/轻量查询 | 采集层(2人)+策略师 | 速览报告 | ~15s |
| **🏛️ 完整模式** | 深度分析/月审/重大决策 | 全员(四层) | 完整投研报告 | ~60s |

### 模式选择逻辑

```
总监接到指令
  ↓
┌─ 含"预警/监控/波动" → ⚡ 预警模式
├─ 含"看看/扫一眼/怎么样" → 🚀 快速模式
├─ 含"分析/深挖/报告" → 🏛️ 完整模式
└─ 默认 → 🚀 快速模式
```

### 应急降级

```
采集层 API 全挂 → 自动降级到「离线模式」
  ├─ 用缓存数据（标注时效）
  ├─ 缺失数据标注"暂不可得"
  └─ 仅输出已有信息，不作综合研判
```

## 四、角色设计模板

### 角色定义

```yaml
角色名: 行情采集员
隶属: 采集层
模式: 完整+快速（预警模式不激活）
职责: 一句话
工具: 具体用什么
产出: 输出格式
独立性: 并行/串行
```

### 不能犯的设计错误

| ❌ | ✅ | 原因 |
|------|------|------|
| 一角色多职责 | 拆成2-3个 | 单一职责 |
| 分析混用方法论 | 各自独立 | 互不污染 |
| 缺质检 | 必有质量官 | 输出把关 |
| 全串行 | 采集分析并行 | 互不等待 |
| 缺HITL | 关键节点确认 | 决策权归用户 |
| 🆕 没有反馈层 | 必有复盘师 | 系统不进化 |
| 🆕 所有任务都全量 | 三档分级 | 轻量场景不浪费 |

### 完整模式流程

```
用户指令
  ↓
总监拆解意图 → 选模式
  ↓
┌───┼───┐
↓   ↓   ↓       采集层并行
A   B   C
↓   ↓   ↓
└───┼───┘
  ↓
核查员（交叉验证）
  ↓
┌───┼───┐       分析层并行
↓   ↓   ↓
X   Y   Z
↓   ↓   ↓
└───┼───┘
  ↓
策略师（综合研判）← HITL 节点
  ↓
风控官 → 质量官
  ↓
交付 ────→ 结果发生 ────→ 🆕 复盘师
                              ↓
                         迭代日志 → 下次运行自动加载
```

### 快速模式流程

```
用户指令 → 采集层(2人) → 策略师 → 一句话输出
```

### 预警模式流程

```
定时触发 → 采集层(1人) → 风控官 → 触发/静默
```

## 六、🆕 分歧裁决机制

当分析层的多份报告互相矛盾时：

| 分歧等级 | 条件 | 处理 |
|:--:|------|------|
| 🟢 轻度 | 得分差<15分 | 策略师自主裁决，取均值 |
| 🟡 中度 | 得分差 15-30分 | 标注分歧，附两方理由 |
| 🔴 重度 | 得分差>30分 或 方向相反 | HITL，暂停请示用户 |

**裁决原则**：有历史胜率→按胜率加权。无历史→按置信度加权。同等→请示用户。

## 七、关键机制速查

| 机制 | 说明 |
|------|------|
| **并行执行** | 同层角色数据互不依赖 → 同时开工 |
| **Reflexion Loop** | 不合格 → 打回 → 最多3轮 → 仍不合格标注 |
| **HITL** | 关键节点暂停等用户确认 |
| **Agent 记忆** | 采集数据缓存，过期前直接复用 |
| 🆕 **反馈闭环** | 交付→结果→复盘→迭代日志→下次加载 |
| 🆕 **模式分级** | 预警/快速/完整三档，按场景选 |
| 🆕 **应急降级** | API 挂→离线模式，不乱编数据 |

## 九、实战案例

### 基金经理 v2.0（4层12人+三档模式）

```
完整模式（全员）
采集层（3并行）→ 核查 → 分析层（3并行）→ 策略→风控→质量
  ↓ 交付
  ↓ 市场走完
反馈层：复盘师 → 迭代日志

快速模式（日常扫盘）
采集(行情+资金) → 策略师 → 一句话速报

预警模式（15:30自动）
采集(行情) → 风控官 → 波动告警/静默
```

### 自媒体团队（4层+三档）

```
完整模式（生成文案）
热点猎手+选题会 → 文案师 → 风格+审核 → 质检 → 数据官
  ↓ 发布
反馈层：复盘师（播放/点赞/评论 vs 预期）

快速模式（找选题）
热点猎手 → 一句话选题建议

预警模式（连断监测）
数据官 → 2天没发 → 督促
```
